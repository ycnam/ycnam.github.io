**\[클릭스트림 데이터 - 우리 사이트에서 뭔일이 벌어지고 있나?\]**

1.  Web Log  
    \-장점: 가장 쉽고 싸게 수집하고 접근할 수 있는 데이터  
    \-단점: 얻기는 쉬운 대신 가공에 많은 노력이 필요. 정확도 낮음.
2.  Web Beacon (1x1의 투명 이미지)  
    \-장점: 구현하기 쉽고 수집할 데이터를 정확하게 지정할 수 있다.  
    \-단점: 개인정보보호이슈, 안티스파이웨어에 의해 자동으로 제거됨.
3.  Javascript Tag  
    \-장점: 웹로그 다음으로 구현하기 쉽고, 정확도가 높다.  
    \-단점: 자바스크립트 비활성 사용자있음. 태그간 충돌 우려  
    \*Google Analytics도 이방식인듯? 
4.  Packet Sniffing  
    \-장점: 사이트 수정없이 방대한 양의 데이터 수집  
    \-단점: 개인정보보호이슈, 높은 비용과 설치의 어려움

**\[경쟁사 데이터 - 남들은 어떻게 되가고 있는지도 알아야지\]**

1.  패널기반 측정 (ex: Comscore)  
    \-장점: 깊이있고 정교한 정보 습득  
    \-단점: 패널의 대표성확보 논란, 샘플집단의 왜곡(직장에선 못씀..)
2.  ISP기반 측정 (ex: Hitwise)  
    \-장점: 샘플규모가 커서 다양한 서핑습관과 사용자층 커버  
    \-단점: 특정 웹사이트에 대해 깊이있는 행동 정보는 알 수 없음
3.  검색 엔진 데이터 (ex: Google Trends)  
    \-장점: 무료(!), 많은 사용자의 행태 대변, 키워드 관련에는 최적  
    \-단점: 분석 규모가 제한적이고 아직 베타 상태. 

**결론 : 각각 장단점이 있으니 그때그때 잘 골라쓰자.**
